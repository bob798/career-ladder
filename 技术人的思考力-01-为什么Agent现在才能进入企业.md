# 为什么 Agent 现在才能进入企业

> 系列：技术人的思考力（一）

## 从一个问题说起

前段时间碰到一个问题：Agent 这个概念不是今天才有的，学术界研究了几十年了，为什么偏偏是最近两三年才开始真正进入企业、进入实际业务？

我把当时的思考整理出来。

## Agent 不是一个新概念

很多人是从 ChatGPT 之后才开始听到"Agent"这个词的，容易觉得它是最近冒出来的新东西。但其实不是。

AI 这个领域从 1956 年达特茅斯会议上被正式提出，最核心的目标就是让机器像人一样完成任务。Agent——也就是"能自主感知环境、做出决策、采取行动的智能体"——一直是这个目标下的核心研究方向。

那为什么研究了几十年，都没有真正走出实验室？

## 过去缺了什么

要让一个 Agent 真正能干活，它至少需要三样能力：

**第一，能理解和推理。** 给它一个任务，它得能听懂你要什么，能把一个复杂的事情拆成步骤，能在信息不完整的时候做判断。

**第二，能动手。** 光想明白了不够，它得能去操作——调一个接口、控制一台设备、执行具体的动作。而且做完之后得能看结果对不对，不对就调整。

**第三，能从经验里学。** 同样的场景碰到第二次，它应该比第一次做得更好。它得能记住之前干过什么、哪些方法管用、哪些踩了坑。

这三样东西，过去各自都有进展，但一直是割裂的——做推理的是一批方法，做控制的是另一批方法，做学习的又是另一套。凑不到一起，Agent 就只能活在论文和实验室里。

## 大模型补上了拼图

2022 年底 ChatGPT 出来之后，事情开始变了。

大模型首先补上的是**推理能力**。这是最关键的一步。过去的 AI，不管是规则系统还是传统的机器学习，处理的都是确定性的、边界清晰的问题。但大模型第一次展现出了通用的理解和推理能力——它能读懂自然语言的意图，能在模糊的条件下做判断，能把一个复杂的任务分解成可执行的步骤。

推理是 Agent 的"大脑"，大脑不行，四肢再发达也没用。这是过去几十年 AI 研究遇到的最关键的瓶颈之一。

紧接着，2023 年前后，**工具调用**的能力跟上了。模型不光能想，还能通过 API 去调用外部工具、操作系统、执行具体的动作。更重要的是，它能在执行之后观察结果，判断有没有达到预期，不行就换一种方式再来——推理和行动形成了循环。

然后是**记忆和经验积累**。从短期的上下文记忆到长期的经验总结，Agent 开始能够记住之前做过的事，能把成功的经验提炼出来复用，遇到类似的问题不用从零开始。

到今天，这三块能力——推理、行动、记忆——在大模型的基础上初步具备了。Agent 才从学术界走出来，变成了能在企业里干活的东西。

## 一个具体的场景

拿一个容易理解的场景来说。

很多行业过去做的"智能化"，说白了是自动化——写规则。某个指标到了阈值就触发某个动作，这种确定性的事情用规则引擎就能做。

但实际业务中大量的决策不是确定性的。变量很多、组合很复杂、不同条件下策略应该不一样。这些事情过去写规则是写不完的。

现在 Agent 能做推理判断，能对接各种系统去执行动作，能从历史数据里学到经验。过去做不了的模糊决策，现在能做了。

这不是因为行业突然需要 AI 了——需求一直都在。是因为技术能力到了这个点上。

## 一点个人的想法

写这篇文章的过程中我意识到一件事：判断一项技术能不能落地，不能只看概念够不够新、论文够不够多。要看它依赖的那几块关键能力有没有到位。缺一块，就差一口气。

Agent 等了几十年，等的就是大模型把推理这块最关键的拼图放上去。一旦到位了，行动能力、记忆能力很快就跟上了，整个事情就通了。

这个思路其实不只适用于 Agent。自动驾驶、具身智能、医疗 AI，你去看每个领域，都能找到类似的"能力拼图"——有些块已经到位了，有些还差。下一篇想展开聊聊这个。
